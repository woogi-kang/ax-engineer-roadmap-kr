# 기여 안내

AX Engineer Roadmap Korea는 기술 이름을 늘리는 것보다 현장에서 검증할 수 있는 판단 기준과 증거를 보강하는 기여를 우선한다.

## 기여할 수 있는 내용

- 빠진 역량·단계·실패 패턴
- 오래되거나 잘못 연결된 출처
- 특정 산업에서 달라지는 제약과 반례
- 조직과 개인을 식별할 수 없게 정리한 배포 사례
- 템플릿 개선
- 오탈자, 깨진 링크, 접근성 개선

## 기여 전에 확인할 것

1. [출처 정책](research/source-policy.md)을 읽는다.
2. 기존 Issue와 Pull Request를 검색한다.
3. 내용이 크게 달라지면 먼저 Issue에서 범위를 합의한다.
4. 회사·고객·동료의 비공개 정보가 없는지 확인한다.

## 로드맵 항목 형식

새 역량 항목은 가능한 한 다음 질문에 답해야 한다.

```markdown
## 역량 이름

### 알기
### 판단하기
### 해보기
### 증명하기
### 실패 패턴
### 출처
```

도구나 강의를 추가할 때는 목록에 이름만 넣지 말고, 어떤 역량을 어떤 방식으로 연습하는 데 필요한지 설명한다.

## 사례 형식

사례에는 최소한 다음 내용이 필요하다.

- 문제와 사용자
- 기존 업무 흐름
- 제약과 비목표
- 가설과 선택
- 구현·배포 범위
- 평가와 실패 처리
- 채택·인수인계
- 결과와 한계
- 재사용할 패턴

실제 성과 수치는 측정 방법과 기간을 함께 제공해야 한다. 공개할 수 없다면 수치를 지우고 관찰 가능한 상태만 남긴다.

## Pull Request

- 한 PR에는 하나의 논점만 담는다.
- 변경 이유와 독자에게 미치는 영향을 설명한다.
- 새 링크에는 확인일과 출처 유형을 기록한다.
- `node scripts/check-links.mjs`와 `node scripts/check-roadmap.mjs`를 실행한다.
- 자동 생성한 초안이라도 제출자가 사실과 문장을 직접 검토한다.

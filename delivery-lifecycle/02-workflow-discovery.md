# 2. 업무 지도

## 목적

문서에 적힌 절차가 아니라 실제로 일이 시작되고 끝나는 흐름, 인수인계, 대기, 예외를 확인한다.

## 핵심 질문

- 무엇이 업무를 시작시키는가?
- 누가 어떤 정보로 판단하는가?
- 같은 정보를 여러 번 복사하거나 확인하는가?
- 일이 가장 오래 멈추는 지점은 어디인가?
- 공식 절차와 실제 절차가 다른가?
- 예외가 발생하면 누구에게 넘어가는가?

## 주요 작업

1. 시작 신호부터 업무 결과까지 실제 순서를 인터뷰·관찰한다.
2. 각 단계의 입력·행동·판단·출력·담당자를 기록한다.
3. 대기, 재입력, 중복 확인, 누락, 비공식 도구를 표시한다.
4. 정상 흐름과 빈도가 높은 예외 흐름을 나눈다.
5. 업무 결과가 다음 부서나 시스템에서 어떻게 사용되는지 확인한다.
6. 현재 지표와 기록이 실제 흐름을 반영하는지 비교한다.

## 산출물

- 현재 업무 흐름
- 역할·시스템·데이터 지도
- 병목과 예외 목록
- 인수인계 손실 목록
- 미확인 질문

## 통과 기준

- 현업 담당자가 흐름을 보고 빠진 단계와 예외를 수정할 수 있다.
- 각 단계의 책임자와 사실의 기준이 연결돼 있다.
- “AI가 처리한다”처럼 내부 판단을 숨긴 단계가 없다.
- 병목이 시간·오류·대기·위험 중 하나의 근거와 연결된다.

## 중단 신호

- 실제 담당자를 만나지 않고 관리자 설명만으로 흐름을 확정한다.
- 공식 매뉴얼을 실제 업무와 동일하게 취급한다.
- 예외가 대부분인데 정상 흐름만 자동화한다.
- 업무의 시작과 종료를 정의하지 못한다.

## 연습

현재 흐름을 Mermaid로 그리고, 담당자 한 명에게 순서·예외·사용하지 않는 단계를 표시해 달라고 요청한다.

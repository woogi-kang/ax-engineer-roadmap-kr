# 3. 프로세스 재설계

## 목적

기존 절차에 AI를 덧붙이기 전에 없앨 일, 합칠 일, 규칙화할 일, 사람에게 남길 판단을 다시 설계한다.

## 분류 기준

| 분류 | 의미 |
|---|---|
| 제거 | 결과에 기여하지 않는 단계 |
| 통합 | 같은 정보와 판단을 반복하는 단계 |
| 표준화 | 입력·출력·기준을 공통화할 단계 |
| AI 보조 | 사람이 판단하되 탐색·요약·초안을 지원 |
| 승인 기반 실행 | AI 제안을 사람이 승인하면 실행 |
| 제한 자동 실행 | 위험과 복구가 통제된 범위에서 자동 실행 |
| 사람 판단 | 맥락·책임·윤리 판단을 사람에게 유지 |

## 주요 작업

1. 현재 단계마다 존재 이유와 사용자를 확인한다.
2. 제거·통합·표준화를 먼저 적용한다.
3. AI가 필요한 단계의 입력·판단·출력을 좁힌다.
4. 정상·예외·수동 대체 흐름을 함께 설계한다.
5. 역할과 승인 지점이 어떻게 바뀌는지 기록한다.
6. 이중 운영 기간과 기존 절차 종료 조건을 정한다.

## 산출물

- 현재/목표 흐름 비교
- 목표 업무 흐름
- 사람·AI 역할표
- 예외·수동 대체 경로
- 기존 절차 종료 계획

## 통과 기준

- 기존 절차를 그대로 자동화하지 않았다.
- 각 AI 단계가 분명한 사용자와 결과를 가진다.
- 사람 승인이 필요한 이유와 기준이 적혀 있다.
- 실패 시 원래 업무 방식으로 돌아갈 수 있다.
- 역할 변화와 기존 절차 종료 조건이 포함돼 있다.

## 중단 신호

- 새 흐름이 기존보다 단계와 책임자를 더 많이 만든다.
- 사람 승인이 책임 회피를 위한 형식으로만 남는다.
- 자동화가 실패하면 업무 자체를 수행할 수 없다.
- 현업 시간이 줄지만 검토·수정 시간이 더 늘어나는 구조다.

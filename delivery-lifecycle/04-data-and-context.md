# 4. 데이터와 맥락

## 목적

AI가 사용할 정보의 출처, 의미, 최신성, 권한을 정하고 원본부터 업무 결과까지 추적 가능한 연결을 만든다.

## 정보 구분

| 종류 | 예시 | 운영 원칙 |
|---|---|---|
| 원본 데이터 | 주문, 계약, 고객 문의, 재고 | 원본 시스템과 소유자 유지 |
| 가공 데이터 | 정제·통합·분류 결과 | 변환 규칙과 버전 기록 |
| 문서 지식 | 정책, 매뉴얼, 회의록 | 적용 범위와 최신성 표시 |
| 업무 상태 | 승인 대기, 처리 중, 완료 | 실시간 원본 시스템에서 확인 |
| AI 결과 | 분류, 요약, 초안, 제안 | 입력·모델·평가·승인과 연결 |
| 실행 결과 | 생성된 작업, 변경된 상태 | 감사 식별자와 되돌림 정보 기록 |

## 주요 작업

1. 사실의 기준이 되는 시스템과 데이터 소유자를 찾는다.
2. 필드·식별자·단위·시간대·결측 의미를 데이터 계약으로 정한다.
3. 업무 용어와 지표의 정의를 맞춘다.
4. 원본·가공·AI 결과·실행 결과를 분리한다.
5. 최신성, 보존 기간, 접근 권한, 삭제 기준을 정한다.
6. 누락·중복·오래된 값·충돌을 평가 데이터에 포함한다.

## 산출물

- 데이터 계약
- 원본 시스템·소유자 지도
- 용어집과 지표 정의
- 데이터 계보
- 품질·최신성·권한 규칙

## 통과 기준

- 중요한 결과가 어떤 원본에서 왔는지 추적할 수 있다.
- 실시간 상태와 문서 검색의 차이가 설계에 반영돼 있다.
- 누락과 표본 편향을 표현할 수 있다.
- 데이터 소유자와 접근 승인자가 확인됐다.
- AI 결과를 원본 사실로 덮어쓰지 않는다.

## 중단 신호

- 원본 시스템을 알 수 없는데 복제 데이터로 결정을 자동화한다.
- 같은 용어가 팀마다 다른 뜻인데 정의를 미룬다.
- 수집 범위와 누락을 숨긴 채 정확도를 계산한다.
- 필요한 데이터 접근 권한을 얻을 현실적인 경로가 없다.

# 5. 실행 계약과 통제

## 목적

AI 업무의 입력·출력·평가·권한·승인·기록·복구를 한 계약으로 묶는다.

## 최소 계약

| 영역 | 확인할 내용 |
|---|---|
| 목적 | 누구의 어떤 업무 결과를 바꾸는가 |
| 입력 | 필수 필드, 출처, 최신성, 누락 처리 |
| 출력 | 스키마, 형식, 품질 기준, 근거 |
| 평가 | 정상·경계·실패 사례와 판정 기준 |
| 권한 | 읽기·제안·생성·변경·외부 전송 범위 |
| 승인 | 승인자, 승인 조건, 만료와 위임 |
| 기록 | 실행·모델·프롬프트·도구·결과 버전 |
| 복구 | 중단·재시도·되돌림·수동 처리 |
| 비용 | 건별·기간별 한도와 초과 행동 |
| 책임 | 업무·데이터·기술·보안 책임자 |

## 자율성 수준

| 수준 | AI 역할 | 실행 |
|---|---|---|
| A0 | 탐색·초안 없음 | 사람이 전부 수행 |
| A1 | 검색·요약·초안 | 사람이 판단·실행 |
| A2 | 선택지와 근거 제안 | 사람이 선택·실행 |
| A3 | 실행안 생성 | 사람 승인 후 시스템 실행 |
| A4 | 제한된 자동 실행 | 정책·감시·복구 범위 안에서 실행 |

자율성은 모델 성능만으로 높이지 않는다. 피해 규모, 탐지 가능성, 되돌림 가능성, 책임 구조를 함께 본다.

## 주요 작업

1. [실행 계약 템플릿](../toolkit/execution-contract.md)을 작성한다.
2. 정상·경계·실패 평가 사례를 연결한다.
3. 역할별 권한과 승인 만료를 정한다.
4. 실행과 결과를 같은 식별자로 기록한다.
5. 실패 유형별 중단·재시도·복구를 정한다.
6. 계약 버전과 호환성 정책을 정한다.

## 산출물

- 실행 계약
- 평가 데이터와 판정 기준
- 역할별 권한·승인표
- 감사 이벤트 스키마
- 실패·복구 절차
- 버전·호환성 정책

## 통과 기준

- AI가 할 수 있는 일과 할 수 없는 일이 모두 적혀 있다.
- 승인자가 무엇을 보고 판단하는지 알 수 있다.
- 성공한 실행도 업무 결과 검토와 연결된다.
- 같은 외부 작업이 중복 실행되지 않는다.
- 실패를 재현하고 수동으로 복구할 수 있다.

## 중단 신호

- 프롬프트가 업무 계약을 대신한다.
- 승인 여부만 기록하고 승인 근거를 남기지 않는다.
- 실행 권한은 있지만 되돌리거나 중단할 권한이 없다.
- 모델·도구 변경이 평가 없이 운영에 반영된다.

# 6. 배포와 운영

## 목적

얇은 프로토타입을 인증·권한·관측성·복구·비용 통제가 포함된 운영 시스템으로 옮긴다.

## 주요 작업

1. 운영과 개발 환경, 설정, 시크릿을 분리한다.
2. 인증·인가와 최소 권한을 구현한다.
3. 입력 검증, 중복 방지, 타임아웃, 재시도를 구현한다.
4. 로그·메트릭·트레이스·감사 이벤트를 연결한다.
5. 품질·지연시간·오류·비용 경보를 정한다.
6. 롤백과 수동 대체 경로를 실행해 본다.
7. 배포·장애·복구·운영 절차를 문서화한다.

## 최소 관측 항목

- 업무·실행 식별자
- 사용자 또는 시스템 주체
- 입력·출력 계약 버전
- 모델·프롬프트·도구 버전
- 시작·종료 시간과 상태
- 승인·실행·복구 이벤트
- 오류 유형과 재시도 횟수
- 지연시간과 비용

민감한 원문과 결과를 무조건 로그에 저장하지 않는다. 추적 가능성과 데이터 최소화를 함께 설계한다.

## 산출물

- 배포된 얇은 수직 기능
- CI/CD와 환경 구성
- 운영 대시보드 또는 상태 보고
- runbook과 장애 대응 절차
- 롤백·복구 훈련 기록

## 통과 기준

- 다른 개발자가 배포 구성을 재현할 수 있다.
- 권한 없는 사용과 시스템 호출이 차단된다.
- 실패 원인과 영향을 같은 식별자로 추적할 수 있다.
- 외부 시스템 장애와 모델 실패를 구분한다.
- 롤백 또는 수동 대체 흐름이 실제로 작동한다.

## 중단 신호

- 운영이 개인 계정과 로컬 환경에 의존한다.
- 장애를 사용자 제보로만 발견한다.
- 모든 오류를 같은 방식으로 자동 재시도한다.
- 비용 한도와 중단 기준이 없다.
- 데이터·보안 책임자가 운영 경계를 확인하지 않았다.

# AX 업무 후보 평가표

## 평가 방법

각 항목을 `1~5점`으로 평가한다. 총점만으로 결정하지 않고 위험·데이터·복구 가능성 중 하나라도 기준에 미달하면 보류한다.

| 항목 | 1점 | 3점 | 5점 | 가중치 |
|---|---|---|---|---:|
| 업무 가치 | 편의 개선 | 팀 생산성·품질 | 핵심 비용·매출·위험 | 3 |
| 빈도 | 드물다 | 주·월 단위 | 매일·상시 | 2 |
| 문제 근거 | 의견만 있음 | 일부 기록 | 기준선과 반복 근거 | 3 |
| 데이터 준비 | 접근·정의 불명 | 일부 정리 필요 | 소유자·계약·접근 확인 | 3 |
| 평가 가능성 | 판정 어려움 | 사람 검토 가능 | 반복 평가 세트 가능 | 3 |
| 복구 가능성 | 되돌리기 어려움 | 수동 보정 가능 | 즉시 중단·복구 가능 | 3 |
| 통합 난이도 | 핵심 시스템 변경 | 일부 연결 필요 | 읽기·제안부터 시작 가능 | 2 |
| 채택 가능성 | 책임자 없음 | 제한 사용자 | 책임자·사용자·운영자 확인 | 3 |
| 재사용 가능성 | 일회성 | 유사 업무 존재 | 여러 업무의 공통 패턴 | 1 |

## 필수 통과 조건

- [ ] 업무 책임자와 사용자가 확인됐다.
- [ ] 현재 상태를 측정하거나 관찰할 수 있다.
- [ ] 데이터 소유자와 접근 경로가 있다.
- [ ] 초기 범위에서 위험한 실행을 제외할 수 있다.
- [ ] 실패를 탐지하고 수동으로 처리할 수 있다.
- [ ] 2~4주 안에 끝까지 연결한 얇은 기능을 만들 수 있다.
